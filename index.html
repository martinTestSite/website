tagger.site

What: A basic website that uses javascript to read header metadata from audio files and generates a tracklisted timestamp.

Why: Before this website, the only way to create these tracklisted timestamps was to manually calculate the tracklengths. A time-consuming task that music-uploaders have been stuck with forever, before tagger.site.


discogstagger.site

What: A ruby-on-rails web app that uses the discogs-api to generate comma sepperated tags based on a discogs release.

Why: Before this website, if you wanted a list of tags to put in a YouTube video, you would have to write them by hand. This site simplifies the process for music uploaders by generating a list of tags based on a release's artists / band members / alliases / tracklist etc...


(upcomming Spring 2019) De-Centralized Playlist

What: An android app / chrome extension / SQL server that tracks user music listening history accross multiple devices (Mobile, Desktop) and platforms (Soundcloud, Spotify, YouTube, files, etc).

Why: Listening data is crucial to generating good analysis, and I want to make listening data more easily obtainable accross platforms.


VLC-Radio

What: A light-weight python script that pulls song metadata and albumart from VLC media player, and saves it to a desired location.

Why: Intended to make the process of displaying what's currently-playing easier for streamers. I use this script multiple times for automated radio streams on YouTube.


VLC-Playlist-Randomizer

What: Python program to randomize a VLC playlist .xspf file.

Why: VLC's built in random-feature jumps around and doesn't shuffle all too well. I built this script so I could generate unique playlists for large amounts of songs to play with my VLC-Radio project.


Simple Shell in C

What: This program executes a simple shell in C, which can run background proccesses, command line arguments, and perform I/O redirection.
